# User Segmentation Guide / 用户分层指南

This guide defines project-level assumptions for user segmentation. These segmentation rules are working hypotheses for this project and can be adjusted later based on business needs and validation results.

本文档定义本项目中的用户分层假设。这些分层规则是项目假设，后续可以根据业务需求和验证结果进行调整。

## Heavy Users / 重度用户

English: Heavy Users are users with high watch frequency, high watch time, or strong content interaction compared with the rest of the dataset.

中文：重度用户是指相对于数据集中其他用户，观看频率较高、观看时长较高或内容互动更强的用户。

## Medium Users / 中度用户

English: Medium Users show moderate watch frequency and engagement. They are neither clearly highly active nor clearly inactive in the project dataset.

中文：中度用户表现出中等水平的观看频率和互动程度，在本项目数据集中既不明显高活跃，也不明显低活跃。

## Light Users / 轻度用户

English: Light Users have lower watch frequency, lower watch time, or weaker content interaction than most users in the dataset.

中文：轻度用户是指观看频率较低、观看时长较低或内容互动弱于多数用户的用户。

## At-risk Users / 高流失风险用户

English: At-risk Users are users whose observed behavior suggests higher churn risk, such as long time since last login, low watch time, low completion rate, or low recommendation click rate.

中文：高流失风险用户是指观察行为显示流失风险较高的用户，例如距离上次登录时间较长、观看时长较低、完播率较低或推荐点击率较低。

## Adjustment Principle / 调整原则

English: Segmentation thresholds should be treated as project assumptions. They should be reviewed with business context, data distribution, and future evaluation results.

中文：分层阈值应被视为项目假设，需要结合业务背景、数据分布和后续评估结果持续调整。
