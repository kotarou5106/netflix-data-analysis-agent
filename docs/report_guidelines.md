# Report Guidelines / 报告撰写指南

These guidelines define how analysis reports should be written in this Netflix Data Analysis Agent project.

本文档定义 Netflix Data Analysis Agent 项目中分析报告的撰写要求。

## Evidence / 证据

English: Every report must include evidence. Evidence should come from data query results, calculated metrics, comparisons, or documented knowledge base definitions.

中文：每份报告都必须包含证据。证据应来自数据查询结果、计算指标、对比分析或知识库中的指标口径。

## Limitations / 限制说明

English: Every report must include limitations. The report should explain data scope, observational nature, metric assumptions, and any uncertainty in the analysis.

中文：每份报告都必须包含限制说明。报告应说明数据范围、观察性分析属性、指标假设以及分析中的不确定性。

## Avoid Causal Claims / 避免因果性表述

English: Do not write correlation as causation. A pattern or association in the dataset should not be described as proof that one factor caused another outcome.

中文：不要把相关性写成因果性。数据中的模式或关联不应被描述为某个因素导致某个结果的证明。

## Business Recommendations / 业务建议

English: Business recommendations must be based on data evidence and the definitions in the knowledge base. Recommendations should be framed as reasonable next steps or hypotheses to test.

中文：业务建议必须基于数据证据和知识库口径。建议应被表述为合理的下一步行动或待验证的业务假设。
